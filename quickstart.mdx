---
title: "Quickstart: Run FLOSS and Extract Malware Strings"
description: "Learn how to extract obfuscated strings from a malware binary with FLOSS in a few simple steps, from installation to your first results."
---

FLOSS works like `strings.exe`, but goes further by automatically recovering strings that malware hides through stack construction, tight loops, and decoding functions. This guide walks you through running your first analysis and interpreting the output.

<Steps>
  <Step title="Install FLOSS">
    Download the standalone binary for your platform from the [releases page](https://github.com/mandiant/flare-floss/releases). No installation required â€” just download and run.

    Alternatively, install via pip (requires Python 3.10+):

    ```bash
    pip install flare-floss
    ```

    Verify the installation:

    ```bash
    floss --version
    ```
  </Step>

  <Step title="Run your first analysis">
    Point FLOSS at any PE or ELF binary. FLOSS extracts all four string types by default:

    ```bash
    floss malware.exe
    ```

    FLOSS outputs results grouped by string type:

    ```
    +--------------------------+
    | FLOSS STATIC STRINGS (42)|
    +--------------------------+
    CreateRemoteThread
    VirtualAllocEx
    WS2_32.dll
    ...

    +----------------------------+
    | FLOSS STACK STRINGS (7)    |
    +----------------------------+
    cmd.exe /c del
    ...

    +-----------------------------+
    | FLOSS DECODED STRINGS (12)  |
    +-----------------------------+
    http://192.168.1.100/update
    HKCU\Software\Microsoft\Windows\Run
    ...
    ```
  </Step>

  <Step title="Filter string types">
    Extract only the string types you care about using `--only`, or exclude types you don't need with `--no`.

    Extract only stack and tight strings (useful when you already know the static strings):

    ```bash
    floss --only stack tight -- suspicious.exe
    ```

    Skip static strings to focus on decoded/obfuscated content:

    ```bash
    floss --no static -- backdoor.exe
    ```

    <Note>
      The double dash (`--`) is required to separate the multi-value flag from the filename. `--no` and `--only` cannot be used together.
    </Note>
  </Step>

  <Step title="Export results as JSON">
    Use `-j` to emit structured JSON output for scripting or tool integration:

    ```bash
    floss -j malware.exe > results.json
    ```

    The JSON output contains a `ResultDocument` with `metadata`, `analysis`, and `strings` sections. You can load these results back later without re-running the analysis:

    ```bash
    floss --load results.json
    ```
  </Step>

  <Step title="View all available options">
    Show the most commonly used arguments:

    ```bash
    floss -h
    ```

    Show all supported arguments including advanced options:

    ```bash
    floss -H
    ```
  </Step>
</Steps>

## Common use cases

| Goal | Command |
|------|---------|
| Full analysis (all string types) | `floss malware.exe` |
| Only decoded/obfuscated strings | `floss --only decoded -- malware.exe` |
| Skip static strings | `floss --no static -- malware.exe` |
| JSON output for scripting | `floss -j malware.exe > out.json` |
| Analyze 32-bit shellcode | `floss -f sc32 shellcode.bin` |
| Minimum string length 8 | `floss -n 8 malware.exe` |
| Quiet mode (one string per line) | `floss -q malware.exe` |
